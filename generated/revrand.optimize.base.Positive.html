revrand.optimize.base.Positive¶

class revrand.optimize.base.Positive(lower=None, upper=None, shape=())¶

Define a positive only bound for the optimiser. This may induce the ‘log trick’ in the optimiser (when using an appropriate decorator), which will ignore the ‘smallest’ value (but will stay above 0).

Parameters:lower (float) – The smallest value allowed for the optimiser to evaluate (if not using the log trick).

Examples

>>> b = Positive()
>>> b 
Positive(lower=1e-14, upper=None)

Since tuple (and by extension its descendents) are immutable, the lower bound for all instances of Positive are guaranteed to be positive.

>>> c = Positive(lower=-10)
